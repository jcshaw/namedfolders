Версия 3.0.0.241 [snapshot]
----------------------------------------------------------------------------
Ключевые изменения
~~~~~~~~~~~~~~~~~~
[!] Плагин переведен на Far API 2.0 и стал юникодным.
[!] Выпущена 64-битная версия плагина. Теперь плагин доступен в
двух вариантах: для x86 и x64.
[!] Для плагина создан open source проект на google code:
http://code.google.com/p/namedfolders/. Сюда можно сообщать об ошибках и
пожеланиях по развитию функциональности. Здесь же можно скачать последний
снапшот плагина (релизы будут попрежнему выкладываться на PlugRing).

Исправления
~~~~~~~~~~~
[-] Исправлен запуск 64-битных приложений из 32-битной версии плагина
(#6, http://code.google.com/p/namedfolders/issues/detail?id=6)
[*] Изменена процедура удаления двойных префиксов. Ранее, 
команда "cD:ab:command" рассматривалась плагином как "cd:command", а 
команда "cd:cd:cd:command" не работала. Теперь плагин игнорирует все
префиксы, кроме последнего, так что команда "ab:cd:ef:command" рассматривается
как "ef:command".
[*] Изменен синтаксис для поиска в локальных директориях. 
Теперь команду "cd:alias.." (искать на один уровень выше, чем директория,
на которую указывает alias) можно записывать более традиционно - "cd:alias\.."
Одновременно поддерживаются и старый ("..") и новый ("\..") синтаксис.

по умолчанию ищутся по маске "a*", а не по маске "*a*" как раньше.
[*] Теперь, при копировании директории на панель плагина, появляется
диалог создания псевдонима и пользователь может отредактировать имя создаваемого

теперь через ".." (ранее через "."). Таким образом, команда cd:f\* ищет все
поддиректории в "f", а команда "cd:f..*" - все поддиректории в директории, 
в которой находится "f". Такая замена позволяет использовать точку при
поиске в локальном пути, что актуально, например, в команде "cd:f\*.exe"
[-] При поиске файлов с указанием локального пути, содержащего

содержались символы "#", "@", "(", ")".
[*] При переименовании псевдонима по Shift+F6, если новое имя 
псевдонима указано без слеша, то слеш подставляется автоматически. Т.е. 
если переименовываем "/@@" в "##", то плагин переименовывает псевдоним в
"/##". Ранее в этом случае вместо псевдонима создавался каталог
[-] Плагин ошибочно использовал фиксированную ширину колонок
на панели, вместо того, чтобы брать значения ширины из настроек по 
умолчанию для текущего режима просмотра.

Версия 2.5 релиз [9.01.2006]
----------------------------------------------------------------------------
[+] Во все меню выбора вариантов добавлен фильтр. Пользователь набирает
английские буквы/цифры, при этом в меню остаются только те элементы, в названии
которых содержится набранная строка. Несколько фильтров задаются через пробел,
при этом в меню показываются только те элементы, которые удовлетворяют всем
фильтрам. Для набора русских букв нужно удерживать SHIFT. 
[+] Новый параметр в настройках "Использовать режим одного меню".
Если этот режим выключен, то команда "cD:abc\cde" последовательно выдает два
меню - вначале меню псевдонимов, удовлетворяющих "abc", затем меню 
директорий, вложенных в выбранную именованную директорию и удовлетворяющих "cde". 
Если же включен, то выдается одно меню - для каждого псевдонима "abc"
находятся все варианты "cde" - и выдается общий список всевозможных директорий.
[-] Убран ряд багов связанных с русскими буквами в именах директорий.
[-] Убран баг в команде "cd: :soft". Если она подавалась из 
директории, содержащей русские буквы, то русские буквы запоминались в 
буфере обмена в неправильной кодировке.
[-] Не устанавливалась горячая клавиша для плагина в меню дисков.
[-] В новых версиях FAR не работали кнопки Ok и Cancel в диалогах.
[-] Поиск локальных путей велся некорректно, если локальный
путь содержал несколько вложенных директорий, причем в названии одной из них 
использовался метасимвол (например, маска a\b\c срабатывала, а "a\b*\c" - нет).
[-] При копировании директории на панель плагина, в названии директории
задваивался слеш, расположенный непосредственно за буквой диска.
[-] Убран баг с зависанием при вызове именованной директории
вида "net:\\abc" из панели плагина. Баг начал проявляться начиная с FAR версии 2060.
[+] Новый параметр в настройках: режим расширения масок в локальных путях.
Для команды cd:sh\path задает способ преобразования названий директорий в маски 
в path. Варианты: 0: a -> *a*. 1: a -> a*; 2: a -> a. Как и раньше преобразование
осуществляется только в том случае, если "a" не содержит метасимволов.
[+] В меню выбора программ (команда cd: soft) добавлены режимы отображения.
[*] Новый макрос FastPrefix.reg, заменяющий FastPanel.reg. Новый 
макрос закрепляет команду вызова панели за двоеточием ":" и буквой
"ж". Таким образом, независимо от текущей раскладки клавиатуры (русская,
английская) нажатие ":" будет приводить к открытию панели. Русская
раскладка поддерживается только в новых версиях Far (1.7 alpha 6)
[*] Убран режим древовидного отображения каталогов.
[*] Состояние именованных директорий теперь проверяется
только в режиме панели Ctrl+8. В других режимах состояние не проверяется,
что положительно сказывается на скорости обновления панели.
[+] В настройки добавлен режим "Субдиректории как псевдонимы". 
Он позволят с помощью плагина искать директории, вложенные в текущую.
В списке псевдонимов субдиректорий отображаются в виде ".\abc".
[*] При переходе к файлу по cf: курсор панели 
устанавливается на выбранном файле

[-] Исправлен баг: при удалении кнопки "Удалить все" и

[*] Подтверждение на удаление теперь общее для псевдонимов
и пустых каталогов. Для непустых каталогов отдельное подтверждение.
[*] Диалог выполнения команды теперь не закрывается, если команда
выполнена неудачно (например, не найден псевдоним).
[-] Команды неявного создания и удаления не работали, если
указывался каталог (например, "cD::cat/")
[+] Новая форма команды "cd:path" - "cd:.relativepath" (с точкой) -

[+] Новая команда "cd:--f sh\path" для поиска файлов удовлетворяющих
path в директории, на которую ссылается псевдоним sh.
[+] Новая команда "cd:--df sh\path" для поиска файлов и директорий,
удовлетворяющих path в директории, на которую ссылается псевдоним sh.
[+] Новые reg-файлы "cs.reg", "cf.reg", "cdf.reg","cc.reg" (в
поддиректории Patterns) - шаблоны для быстрого доступа к командам 
"cd: :soft", "cd:--f ", "cd:--df ", "cd:.\"

Функциональность и удобство использования
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
[+] Команду "cd: soft" теперь можно вызывать с параметрами.
Указанные параметры будут переданы запускаемой программе. Команда 
пригодится в случае, когда нужно открыть из FAR заданный файл с 
помощью определенной программы. Пример: команда "cd: word abc.txt"
позволит запустить MsWord и открыть в нем текстовый файл.

[!] Теперь минимальными системными требованиями для плагина 
являются Win9X/WinNT + IE4.0. Плагин был протестирован под Win98.
[+] Новая команда "cd:path". В качестве path должен быть
указан абсолютный локальный путь. Фактически, такая команда позволяет
использовать в качестве псевдонима имя диска. Пример: команда "cd:z:\\a*"
выведет список папок первого и второго уровня на диске z, начинающихся
на букву a. 
[+] На сайте плагина появилась страничка TODO:
http://www.rammus.ru/?tag=nf_todo 
[+] Теперь при удалении нескольких псевдонимов и/или
каталогов, с виртуальной панели, вместо того, чтобы подтверждать
удаление каждого элемента в отдельность, можно дать команду
"Удалить все". Команда "Отмена" теперь отменяет процесс удаления,
а не удаление отдельного элемента, как раньше.
[*] В команде "cd:sh" многобуквенный префикс в названии
псевдонима игнорируется. Пример: комадна "cd:cd:p" будет обработана
как "cd:p".


Исправление ошибок
~~~~~~~~~~~~~~~~~~
[-] При создании папок, названия папок на русском языке сохранялись
крокозябриками.
[-] В меню выбора папок неверно отображались папки с амперсендами
в названиях.
[-] Не всегда срабатывали маски, содержащие классы символов "[]".
Баг проявлялся, если других метасимволов в маске не было и, кроме того, 
в качестве класса символов указывалось более одного символа. Пример: 
С:\Tasks\[TD]o[dn][oe].
[-] Команда "cd:path" теперь обрабатывается как "cd:*path*",
если path не содержит метасимволов и как "cd:path*", если содержит. Т.е.
появилась возможность искать папки по первым буквам названия.
[-] При создании папок курсор теперь не перескакивает в 
начало списка, а позиционируется на созданном элементе. 
Аналогично, при обновлении панели сохраняется позиция курсора.

[+] Новая команда "cd: soft" (префикс, пробел, шаблон) для запуска программы, 
ярлычок на которую имеется в меню "Пуск". По этой команде плагин находит в 
"Пуск->Программы" все  ярлыки, названия которых удовлетворяют 
шаблону "*soft*", предлагает выбрать  требуемый ярлык из списка и запускает 
его. Если "soft" представляет из себя маску (например, "studio*"), то 
поиск осуществляется по этой маске, без добавления звездочек. 
[+] Новая команда - экспериментальная. Предложения по юзабилити 
приветствуются :)

Версия 2.2 релиз [15.04.2004]
----------------------------------------------------------------------------
Поддержка метасимволов в путях
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
[+]  Добавлена возможность регулировать глубину поиска в путях к именованным
локальным директориям и в относительных путях. Глубина поиска регулируется
количеством слешей. N слешей означает: "искать директорию во всех директориях, 
вложенных не более чем на N уровней относительно текущей". Т.е:
"b\a"  - найти "a" в директориях, непосредственно вложенных в "b".
"b\\a" - найти "a" в директориях, непосредственно вложенных в "b" и в 
их поддиректориях.
 и т.д.

[+]  Добавлена возможность использовать метасимволы в путях к локальным 
именованным директориям и в относительных путях. Поддерживаются стандартные 
метасимволы FAR:  "?", "*" и символьные классы ("[a-b, z]"). 
Действие "*" НЕ распространяется на вложенные директории. Следует явно задавать 
глубину поиска количеством слешей.

[!]  Примеры использования новых возможностей:
 Для всех директорий Program Files (расположенных на разных дисках) можно
     задать общий псевдоним: "pf" = "?:\Program Files" или "[c-d, f]:\Program Files".
 Появился быстрый способ перехода в директорию, расположенную где-то в 
     недрах именованной директории:  "cd:pf\office", "cd:pf\\vc", "cd:pf\\delphi", 
"cd:pf\\\atl" и т.д.


Поддержка системных директорий
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
[+]  Теперь в качестве именованной директории можно указать ключ реестра,
в виде "$HKEY_LOCAL_MACHINE\\Software..." (первым символом должен быть "$").
При вызове такого псевдонима, плагин просматривает указанный ключ реестра,
составляет список значений переменных этого ключа (учитываются только
значения, содержащие пути к существующим директориям) и предлагает 
пользователю выбрать требуемое значение из списка. 

[!]  Указанная возможность позволяет посредством одного псевдонима  
обращаться ко всем стандартным  директориям системы:
"My documents", "History", "Favorites" и т.д.

[+]  В дистрибутив плагина добавлен файл "StdPaths.reg", демонстрирующий
применение новой возможности. Файл создает два псевдонима: "stda" и "stdu",
соответственно, для доступа к стандартным директориями объявленных для
всех пользователей и для текущего пользователя. 


Меню вариантов
~~~~~~~~~~~~~~
[+]  Доработано меню выбора псевдонима. Теперь по нажатию F11 переключается
режим отображения каталогов. Поддерживается два режима: в первом 
(стандартном) каталог пишется рядом с каждым псевдонимом; 
во втором псевдонимы вложенные в один и тот же каталог объединяются 
в группу, причем название каталога пишется только для первого псевдонима 
группы. По идее, такой режим напоминает древовидный...

[+]  Изменен способ открытия псевдонимов, указывающих на две директории.
Теперь директория, указанная первой, всегда открывается на левой панели,
указанная второй - на правой. Если пользователь открывает псевдоним
на неактивной панели (удерживая Ctrl), то директории открываются в обратном 
порядке.

[+]  Shift+Del удаляет каталоги и псевдонимы.

Командная строка
~~~~~~~~~~~~~~~~
[+] Поддержка метасимволов в именах псевдонимов, каталогов и директорий.
[!] В командах удаления и открытия псевдонима, имя псевдонима в 
общем виде записывается как [каталог/][псевдоним][\путь относительно 
именованной директории]
[+] Новая команда для создания псевдонима сразу для двух директорий. 
Команды "cd:::псевдоним" и "cd::+псевдоним" создают, 
соответственно, постоянный и временный псевдоним сразу для двух директорий,

[+] Новые команды для неявного создания псевдонимов: "cd::", "cd:+",
"cd:::", "cd::+". В качестве имени псевдонима 
принимается имя директории,открытой на активной панели.
[+] Новая команда для неявного удаления псевдонима "cd:-" - 
выводит меню со списком псевдонимов, ссылающихся на директорию, открытую

приводит к его удалению.
[+] Новая команда для поиска псевдонима по директории, на которую 
он ссылается: "cd:~mask" находит все псевдонимы, которые 
ссылаются на директории, путь к котором удовлетворяет маске "*mask*". 

[+] Команду плагину можно теперь подавать через
"Диалог выполнения команды" (F11 -> Named Folders 2.0 -> Execute command).

в диалоге префикс плагина ("cd:") можно не указывать. Кроме этого, 
диалог позволяет просматривать историю команд (опционально). Вместе с 
плагином поставляется файл "ApplyCommand.reg", который присваивает 
команде вызова диалога горячую клавишу "Ctrl+D".

[*] Поддерживается поиск среди псевдонимов по Alt+F7
